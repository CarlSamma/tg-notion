## Cosa è cambiato

### Tag — flusso completamente ridisegnato (`tagger.py` + `state.py` + `handlers.py`)

La keyboard ora ha **tre sezioni distinte**:

```
[✅ machinelearning]  [✅ python]      ← tag AI (pre-selezionati, togglabili)
──── libreria ────
[🏷️ startup]         [🏷️ design]      ← top tag libreria (off, selezionabili)
[🏷️ politica]
[✏️ Aggiungi tag...]                   ← apre input testo libero
[✅ Archivia  (4 tag)]                 ← conta live i tag totali
```

L'utente preme "✏️ Aggiungi tag..." → bot risponde chiedendo il testo → l'utente scrive `#ai #llm` oppure `ai, llm` oppure `ai llm` → i tag vengono aggiunti e mostrati nella keyboard. Tutto senza ricaricare il flusso.

Ordine finale garantito: **tag AI → tag libreria selezionati → tag custom**. Deduplicati, max 15.

### Categoria (`summarizer.py` + `notion.py`)
L'AI ora classifica ogni contenuto in una delle 12 macro-categorie (Tecnologia, AI, Business, Politica, Scienza, Design, Economia, Salute, Cultura, Sport, Intrattenimento, Altro). Diventa un campo `select` nel database Notion — filtro potentissimo per navigare l'archivio.

### Note vocali (`extractor.py` + `handlers.py`)
Il bot accetta messaggi vocali Telegram. Se è impostata `OPENAI_API_KEY`, usa Whisper per trascrivere. Poi la trascrizione va in pipeline normale (sintesi AI + tag + archiviazione). Senza `OPENAI_API_KEY` archivia un placeholder segnalando il gap.

### Comando `/status` (`notion.py` + `handlers.py`)
Invia `/status` al bot e ricevi: totale contenuti archiviati, top 5 tag, top 3 categorie, titolo dell'ultimo archiviato.

### Comando `/help`
Guida completa direttamente nel bot.

### Error recovery (`handlers.py`)
Se l'archiviazione Notion fallisce, il bot mostra titolo + bullet points in chat — il contenuto elaborato non va perso.

### `OPENAI_API_KEY` — nuova variabile opzionale
Aggiungila in Railway per abilitare Whisper. Senza di essa tutto funziona, le note vocali vengono archiviate senza trascrizione.
